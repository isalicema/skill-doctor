# Skill 体检检查清单（判定用词参考）

> skill-doctor 出报告时查这里，统一判定口径。

## 关卡 1：谁触发（入口设计）

| 检查项 | ✅ 健康 | ⚠️ 有问题 |
|--------|--------|----------|
| 触发时机 | 自动/手动定位和「频率×风险」匹配 | 高影响却设成自动触发；或高频却要手动 |
| 触发语一致性 | description 触发语 = 实际用的触发语 | 真实触发语只在 CLAUDE.md/别处，SKILL.md 没写 |
| 上下文成本 | 自动触发 Skill 数量可控 | 一堆低频 Skill 都自动触发，堆满描述 |

**分类口诀**：高频·低风险·边界清 → 自动；低频·高影响·需判断时机 → 手动。

## 关卡 2：结构（步骤 vs 参考材料）

| 检查项 | ✅ 健康 | ⚠️ 有问题 |
|--------|--------|----------|
| 主文件长度 | 十分钟内读完 | 又长又杂，读不完 |
| 内容纯度 | 主文件只有步骤 + 分支指向 | 模板/样例/glossary/长清单焊在步骤里 |
| 参考材料位置 | 只在某情况用的拆到旁边文件 | 单分支才用的东西塞满主文件 |

**自检法**：某段只在一种情况用 → 挪出去；每次都用 → 留主文件。

## 关卡 3：删减（deletion test）

对每段问一句：**「删掉它，Agent 行为会不会变？」**

| 判定 | 含义 | 处置 |
|------|------|------|
| ✅ 保留 | 删了行为会变 = 有效 leading 句/护栏 | 留，别动 |
| 🔶 重复 | 和别处说同一件事 | 合并成一处 |
| 🔶 沉积物 | 没人敢删的旧话、过期内容 | 挪走或删 |
| ❌ no-op | 看着有用、删了行为不变 | 删 |

**保护名单**：能压住整套做法的稳定短词/短句（leading words）不许误删——它们短，但撑着行为。
