# 体检报告模板

> skill-doctor 出报告时套这个结构。用表格，别写成大段散文。

```markdown
## 🔬 <skill-name> Deletion Test 体检报告

### ① 入口设计 —— <✅健康 / ⚠️有问题>
- 现状：<自动还是手动触发，触发语在哪>
- 发现：<触发语是否一致、上下文成本、隐性沉积物>

### ② 结构：步骤 vs 参考材料 —— <✅ / ⚠️>
| 内容块 | 类型（步骤/参考材料） | 是否该留主文件 |
|--------|--------------------|--------------|
| ... | ... | ✅留 / ❌拆出去 |

- 主文件能否十分钟读完：<结论>

### ③ Deletion Test：逐段删删看 —— <发现几处>
| 内容 | 删掉后行为会变吗？ | 判定 |
|------|------------------|------|
| ... | 会变/不变 | ✅保留 / 🔶重复 / ❌no-op |

---

## 📋 体检结论（三句话）
1. **入口**：<一句>
2. **结构**：<一句，通常是最该动手的地方>
3. **文本**：<一句，有没有 no-op / 重复>

**改造优先级**：拆参考材料 > 收敛入口 > 合并重复句 > 删 no-op（按目标 Skill 实际情况排）
```

## 出报告的语气
- 诚实但不苛刻：写得好的 Skill 就明说"这个比我想象的好"。
- 用 deletion test 的话术，别用"这句写错了"——说"删掉后行为不变"，回到结果讨论（Pocock：帮团队少吵架）。
